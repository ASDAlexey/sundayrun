### Идеи на потом

- Sitemap с URL всех опубликованных забегов (генерация при публикации)
- Prerender/SSG для глубокой индексации протоколов
- Динамический `document.title` на страницах забега и атлета (номер/дата, имя)
- Верификационные метатеги Google/Яндекс после регистрации в консолях
- **Хранение данных** — план в [SQLITE_DB.md](./SQLITE_DB.md):
  - Шаг 0: версионный указатель `data/version.json` + SHA-pinned URL jsDelivr (мгновенное обновление у всех читателей вместо ненадёжного purge) + кэш в IndexedDB по SHA
  - SQLite `data/protocol.db` в репозитории + HTTP Range-запросы из браузера (`sqlite-wasm-http`) — точечные SQL-запросы вместо загрузки монолитных `index.json`/`athletes.json` целиком
  - Зеркало репозитория на российском git-хостинге (GitVerse/GitFlic) через GitHub Action + фолбэк-цепочка источников данных на фронте — страховка от блокировок GitHub/jsDelivr для читателей в РФ
