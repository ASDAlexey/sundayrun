# GitHub-токен организатора

Режим организатора (загрузка забегов, публикация протоколов) открывается после ввода
fine-grained GitHub-токена на странице **`/admin`** («для организатора» в футере сайта).
Токен нужен только организатору — обычные посетители сайта читают архив анонимно.

## Как создать токен

1. Войдите на GitHub под аккаунтом, у которого есть **право записи** в репозиторий `protocols`
   (владелец репозитория или collaborator с ролью Write).
2. Откройте <https://github.com/settings/personal-access-tokens/new>
   (Settings → Developer settings → Personal access tokens → **Fine-grained tokens** → Generate new token).
3. Заполните форму:
   - **Token name** — любое понятное имя, например `parkrun-publish`.
   - **Expiration** — рекомендуем 90 дней (максимум — 1 год; по истечении создаёте новый).
   - **Resource owner** — владелец репозитория `protocols` (`ASDAlexey`).
   - **Repository access** — **Only select repositories** → выберите только `protocols`.
   - **Permissions → Repository permissions → Contents** — **Read and write**.
     Больше ничего не включайте.
4. Нажмите **Generate token** и сразу скопируйте значение — GitHub показывает его один раз.
5. Откройте на сайте страницу `/admin`, вставьте токен в поле «GitHub-токен» и нажмите «Сохранить».
   Приложение проверит доступ к репозиторию `protocols` и включит режим организатора.

## Где хранится токен и насколько это безопасно

- Токен сохраняется **только в вашем браузере** (localStorage) на этом устройстве.
  На сервер он не отправляется — сервера у проекта нет; токен уходит только в `api.github.com`
  при публикации.
- Токен ограничен одним репозиторием и одним правом (Contents read/write) — даже при утечке
  им нельзя тронуть другие репозитории или настройки аккаунта.
- Кнопка «Удалить токен» на `/admin` стирает его из браузера.
- Отозвать токен целиком можно на GitHub: Settings → Developer settings →
  Fine-grained tokens → Revoke.

## Частые ошибки

| Сообщение на `/admin`                                    | Причина                                                               | Что делать                         |
| -------------------------------------------------------- | --------------------------------------------------------------------- | ---------------------------------- |
| «Токен не подходит: нет доступа к репозиторию protocols» | Токен без права Contents read/write на `protocols`, истёк или отозван | Создайте новый токен по шагам выше |
| «Не удалось проверить токен»                             | Сеть/GitHub недоступны                                                | Повторите попытку позже            |
