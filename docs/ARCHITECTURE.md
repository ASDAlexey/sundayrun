# Архитектура

Веб-приложение клуба «Воскресный парковый пробег» (Таганрог, КЛБ «Легенда»): организатор загружает Excel-экспорт секундомера → приложение определяет пол участников, считает места, генерирует PDF-протокол по образцу `assets/20.09.2020.pdf` и публикует всё в GitHub. Сайт показывает архив протоколов, а из накопленной истории автоматически считаются примечания.

## Жёсткие требования

- **Всё 100% бесплатно**: без бэкенда, без платных сервисов. Бесплатные OSS-пакеты разрешены.
- Вся логика — на клиенте (браузер). Хостинг и хранилище — GitHub.
- UI: русский приоритетный (source locale `ru`), английский — перевод.
- Стек и дизайн — как в проекте `portfolio`: Angular 22, zoneless, signals, дизайн-система Litely, Bun, Vitest (покрытие 100%).

## Поток данных

```
xlsx (экспорт секундомера)
  → xlsx-reader (fflate + DOMParser)          — ZIP+XML → string[][]
  → timer-export-parser                       — → Participant[] (скип NOTE!, DNF, круги)
  → gender-inference                          — пол по имени (словарь + эвристики) + ручная правка
  → protocol-builder (+ notes-builder)        — места по полу, авто-примечания из истории
  → protocol-doc-definition → pdfmake         — PDF (PT Serif, кириллица) → Blob
  → github-storage (Git Data API)             — атомарный коммит данных в репозиторий sundayrun
```

## Входные данные (проверено на реальных файлах)

- xlsx из LibreOffice, один лист, колонки A–F: `Name, Total, Avg/lap, Avg/km, Lap 1, Lap 2`.
- Времена — **текст** вида `0:19:03,028`. В конце листа мусорный блок «NOTE!» — пропускается.
- **Колонки пола нет** — пол определяется по имени (второй токен «Фамилия Имя») со степенью уверенности; неуверенные строки подсвечиваются для ручной правки.
- Семантика кругов: 2 круга → финишировал 5 км (круг 1 = отсечка 2,3 км); 1 круг → бежал только 2,3 км; нет времени → DNF.
- Метаданных события (№, дата, место) в xlsx нет — вводятся в форме.

## Выходной PDF

A4 «ПРОТОКОЛ» по образцу: шапка (дата, «Воскресный парковый пробег № N», город, парк, клуб), вводный абзац, таблица «Участники» (№, Спортсмен, Время 2,3/5 км, Пол, Место М/Ж, Клуб, Примечание), сокращения DNF/DSQ, подпись председателя. Шрифт PT Serif (OFL, кириллица).

## Выбор библиотек

| Задача                  | Выбор                                                     | Почему                                                                                                                                                                                   |
| ----------------------- | --------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Распаковка/парсинг xlsx | **fflate** (~8 КБ) + нативный DOMParser, свой мини-парсер | Формат фиксированный; SheetJS CE заморожен с CVE, ExcelJS тяжёлый. Парсер за интерфейсом — заменяем одним файлом                                                                         |
| PDF                     | **pdfmake** + PT Serif                                    | Декларативный doc-definition = чистая тестируемая функция; colSpan/rowSpan для двойных шапок; `getBlob()` для скачивания и Web Share. Динамический импорт — не попадает в initial bundle |
| Состояние               | Angular signals (свой store-сервис)                       | Достаточно для мастера из 3 шагов                                                                                                                                                        |

## Хранилище: папка `data/` в репозитории приложения `sundayrun`

Код и данные живут в одном публичном репозитории (решение пользователя, 2026-07-05):
данные — под `data/`, пути собраны в `protocols-repo.constant.ts` (`DATA_DIRECTORY`).
Публикация протокола коммитит только `data/**` — CI-деплой такие пуши пропускает
(`paths-ignore`), сайт читает данные через jsDelivr, а не из сборки.
Компромисс: PAT организатора имеет запись во весь репозиторий, включая код.

```
data/
  index.json                  # манифест событий: [{slug, date, number, title, stats, files}]
  athletes.json               # сводная история: по каждому человеку все забеги,
                              # лучший результат за всё время и по годам
  site-meta.json              # объявление организатора: время старта + сообщение на главной
  events/2026-06-14/
    source.xlsx               # исходный Excel
    protocol.pdf              # готовый протокол
    results.json              # машиночитаемые результаты забега
```

Импортированная история (249 забегов с 2019 г.): сезон 1 — из текстовых PDF
`assets/parkrun-pdfs/`, сезон 2 — с фото-протоколов VK (PDF собраны в
`assets/parkrun-pdfs/результат/`). У импортированных событий нет `source.xlsx`.
Нет протоколов: № 258 (06.11.2022), № 2.60 (30.11.2025), № 2.66 (11.01.2026).

- **Запись**: из браузера через GitHub **Git Data API** (blobs → tree → commit → update ref) — все файлы события + обновлённые `index.json`/`athletes.json` в **одном атомарном коммите**; конфликт ref → refetch + retry. Организатор один раз вставляет fine-grained PAT (scope: только репозиторий `sundayrun`, Contents read/write); хранится в localStorage; при 401 — баннер «токен истёк».
- **Чтение** (архив, история): без токена через **jsDelivr CDN** `cdn.jsdelivr.net/gh/<owner>/sundayrun@main/data/...`, fallback — GitHub Pages этого репозитория. `raw.githubusercontent.com` не используем — блокируется в РФ. После публикации — purge jsDelivr или ссылка на свежий commit SHA. Публичные страницы читают анонимно: `index.json` (`archive.service.ts`), `athletes.json` (`athletes.service.ts`) и `events/<slug>/results.json` (`results.service.ts`); два последних кэшируются на сессию (promise-кэш, сброс при ошибке — перезагрузка страницы повторяет запрос). CDN может отдавать данные с небольшой задержкой после публикации — страницы честно говорят об этом в пустых состояниях и «не найдено».
- **Авто-примечания** считаются из `athletes.json` без обхода всех событий: «Первое участие» (ключа нет в истории), «ЛР (было X)» (новый лучший за всё время), «Лучший результат YYYY г.» (лучший в году, но не абсолютный рекорд). На `/preview` кнопка «Подставить примечания из истории» загружает `athletes.json` через **авторизованный Contents API** (`github/history.service.ts`), а не через jsDelivr: при серийной загрузке старых файлов CDN отдал бы устаревшую копию. Кнопка перезаписывает все примечания (в т.ч. ручные) — действие явное, после него можно править дальше.
- История копится с первого загруженного файла; предусмотрен режим доимпорта старых результатов задним числом.
- **Объявление на главной**: организатор на `/admin` задаёт время старта и сообщение (например, про бонус от партнёра); `site-meta.json` коммитится тем же Git Data API одним файлом (`publish-site-meta.ts`, `site-meta.service.ts`) с purge CDN-пути. Главная читает файл анонимно с jsDelivr и показывает блок над списком забегов; 404/403 или сетевая ошибка — блок просто скрыт.

### Доимпорт старой истории

Старые забеги публикуются **от старых к новым**: примечания каждого события считаются из истории, накопленной предыдущими публикациями. Перепубликация той же даты идемпотентна — событие в `index.json`/`athletes.json` перезаписывается. **Ограничение v1**: при перепубликации старой даты примечания считаются от полной текущей истории, включая более поздние забеги (а не от среза «до этой даты»). Для клубного архива это осознанно допустимо; строгий пересчёт среза — кандидат в v2.

### Локализация контента

PDF-протокол, авто-примечания («Первое участие», «ЛР…») и текст ВК-анонса **не локализуются** — это русскоязычные документы клуба, их формулировки фиксированы независимо от языка UI. Интерфейс размечен под i18n (source locale `ru`); en-локаль отключена 2026-07-05 по решению пользователя — разметка `i18n`/`$localize` сохранена, вернуть перевод можно, добавив локаль в `angular.json` и файл переводов.

## Шаринг и публикация в ВК

1. Мобильные: Web Share API Level 2 — `navigator.share({ files: [pdf] })`, системное меню само показывает Telegram/VK/WhatsApp/MAX.
2. Десктоп/fallback — ссылки на протокол в архиве: `t.me/share/url`, `vk.com/share.php`, `wa.me`, `max.ru/:share`.
3. Всегда доступны «Скачать PDF» и «Скопировать ссылку».
4. **Публикация забега в ВК (админ)**: после генерации приложение составляет мини-описание — № пробега, дата, место, число участников, победители М/Ж (редактируемое). На мобильном PDF + текст уходят через системное меню прямо в приложение ВК; на десктопе — `vk.com/share.php` со ссылкой на протокол + кнопка «Скопировать описание» (ВК не принимает файлы через share-URL — PDF прикрепляется вручную или открывается по ссылке из архива).

## Происхождение данных

На старте забега организатор пользуется приложением-секундомером: на финише каждого бегуна делается отсечка времени. Приложение экспортирует отсечки в xlsx (Name, Total, Avg/lap, Avg/km, Lap 1, Lap 2) — этот файл и загружается на сайт.

## Структура кода

```
src/app/
  core/                              # чистый TS без Angular — юнит-тестируемое ядро
    models/                          # интерфейсы и enum'ы (as const + union type)
    time/duration.ts                 # целые мс, без float
    xlsx/                            # reader + parser + __fixtures__/ (реальные файлы)
    gender/                          # словарь имён + инференс
    protocol/                        # построитель строк протокола
    history/                         # ключ спортсмена, авто-примечания, свод истории
    pdf/                             # doc-definition (чистая) + сервис pdfmake
    github/                          # Git Data API storage (чистые функции: commit, index, token check, jsDelivr)
    share/                           # Web Share + URL-фолбэки
    state/                           # signal store
  github/                            # Angular-сервисы поверх core/github: admin-token (PAT в localStorage), github-storage (публикация), archive (чтение index.json с CDN), athletes (athletes.json с CDN, кэш на сессию), results (results.json события с CDN, кэш по slug), history (athletes.json через Contents API для авто-примечаний)
  features/races | race | athletes | athlete | admin | upload | preview | result    # lazy standalone страницы; admin = вставка токена, races = главная
  shared/ui/                         # stepper, button и т.п. (Litely, BEM)
```

**Главная страница `/` — список забегов, отсортированный от свежего к старому** (самый свежий сверху): дата, № пробега, число участников, «Протокол онлайн» (`/races/<slug>`), просмотр и скачивание PDF, шаринг.

**Публичные страницы (маршруты без guard'ов).**

- `/races/:slug` — онлайн-протокол забега из `results.json`: шапка события, таблица как в PDF (№, спортсмен, время 2,3/5, пол, места М/Ж, клуб, примечание), ссылка на PDF; имя спортсмена ведёт на его страницу. Некорректный slug (не `YYYY-MM-DD`) не доходит до CDN — сразу «не найдено».
- `/athletes` — список атлетов из `athletes.json`: поиск по имени (нормализация как у ключей: регистр, пробелы, «ё»), сортировка по лучшему времени (5 км) и по числу участий.
- `/athletes/:key` — страница атлета: участия/финиши, лучшее время за всё время и по годам (только 5 км), все забеги с фильтрами по году и дистанции и сортировкой по дате/времени; дата забега ведёт на его онлайн-протокол.

**Роли.** Обычные посетители — только просмотр (список, PDF, онлайн-протоколы, страницы атлетов). **Админ** (организатор, у которого в localStorage сохранён GitHub-токен) видит сверху главной блок «Загрузить забег» — вход в мастер `/upload → /preview → /result` (guards не пускают на шаг без данных и без токена). Вход в админ-режим: ссылка «для организатора» → страница вставки fine-grained PAT. Никакого сервера аутентификации не нужно: право записи даёт сам токен GitHub.

## Правила проекта

- Компоненты: standalone, OnPush, `signal()`/`computed()`, `inject()`, `#private` в сервисах, `readonly` наружу.
- `.claude/rules/ts-files.md`: константы → `.constant.ts` (UPPER_SNAKE_CASE), enum'ы → `as const` + union в `.enum.ts`, интерфейсы → `.interface.ts`, типы → `.type.ts`.
- `.claude/rules/spec-files.md`: моки в `.mock.ts`, минимум `it()`-блоков при 100% покрытии, без `async beforeEach`.
- SCSS: только CSS-переменные (hex запрещён), типографика только через миксины `t.text()`, BEM `block__element_modifier`.
- Время — только целые миллисекунды; системные часы в доменной логике не читаются (даты приходят ISO-строками).
- Коммиты — только вручную пользователем.

## Процесс разработки

Мультиагентный конвейер на каждую фазу: исполнитель реализует → adversarial-ревьюер проверяет код и гоняет тесты → фиксер правит по замечаниям → финальный гейт (`check:code-quality` + `test` + `madge`). Статусы фаз — в [ROADMAP.md](./ROADMAP.md).
